🔎 Analyst Response Guide — Password Spray
Phase 1 — Immediate  ⏱ 0–15 min
Step 1 — Verify the spray is real
Why: Confirm before taking disruptive action. A spray is defined by many accounts targeted from the same source IP in a short window — not a single failed login.
  • Check the Source IP column above — one IP hitting multiple accounts is the key spray indicator.
  • Check IP reputation at VirusTotal or AbuseIPDB — known Tor/VPN exit nodes confirm a spray campaign.
  • Open Entra Risk Detections to check whether additional accounts beyond this alert were also targeted.
Step 2 — Revoke sessions on confirmed compromises
Why: An account that had a successful sign-in after failed attempts is likely compromised. The attacker has an active session right now and must be cut off immediately.
  • Click Sign-in Logs for each user above → filter by Status = Success near the detection timestamp.
  • For any confirmed post-spray successful login: click User Profile → Revoke sessions button. This kills all refresh tokens within 1 minute.
  • Reset the account password before the user re-authenticates to prevent re-entry with the compromised credential.
  • Sign-in log scoping query: In Entra sign-in logs filter userId = [affected user], Status = Success, time range = last 24 hours. A successful login from a different IP immediately after a burst of failures is the definitive compromise indicator — this is how you distinguish a breached account from one that was merely targeted.
Phase 2 — Containment  ⏱ 15–60 min
Step 3 — Block the attacking IP(s)
Why: Without blocking the source, the attacker rotates target accounts and continues spraying. The same IP is often used for hours or days across multiple tenants.
  • Entra ID → Security → Conditional Access → Named Locations → + IP ranges → add source IP as a blocked location.
  • Also block at the network/firewall layer for immediate effect — CA policy propagation can take a few minutes.
  • If the IP belongs to a VPN/proxy ASN (check AbuseIPDB), consider blocking the full ASN range in CA.
  • Block by user agent: In Entra sign-in logs, filter for the spray source IP and note the User-Agent string — automated spray tools use distinctive values (e.g. python-requests/2.x, Go-http-client/1.1). Add the user agent as a filter in CA or block it at your WAF/network perimeter to neutralise the same tool even when the attacker rotates IPs.
Step 4 — Enforce MFA on all targeted accounts
Why: Even if the password was not successfully guessed, the targeted accounts remain vulnerable to a second spray or credential stuffing from leaked databases.
  • Confirm all targeted accounts are covered by a CA policy requiring MFA at sign-in.
  • Prefer phishing-resistant MFA (FIDO2 key, Windows Hello, Certificate-Based Auth) over SMS/voice, which can be bypassed with SIM-swapping or real-time phishing proxies.
Phase 3 — Post-Compromise Investigation  ⏱ 1–4 hrs
Step 5 — Hunt for attacker activity on breached accounts
Why: Once inside, attackers set up persistence before any alert fires. They may have already exfiltrated data, created forwarding rules, or consented to a malicious OAuth app.
  • Inbox rules: Outlook → Settings → Rules — look for rules that forward, redirect, or delete emails.
  • OAuth consents: Entra → Enterprise Applications → filter by user — any app consented to after the detection time is suspicious.
  • New registered devices: Entra → Users → [user] → Devices — an attacker-registered device can obtain a PRT and survive a password reset.
  • MFA method changes: Entra → Users → [user] → Authentication methods — any new method added after the detection timestamp is an attacker backdoor.
Step 6 — Harden against recurrence
Why: Password sprays succeed because many environments lack MFA, use weak lockout policies, or allow commonly sprayed passwords.
  • Entra → Security → Authentication methods → Password protection — enable Smart Lockout (threshold: 5 failures, duration: 60 sec).
  • Deploy Microsoft Entra Password Protection to block commonly sprayed weak passwords.
  • Enforce MFA via Conditional Access for all users — not just privileged accounts.
